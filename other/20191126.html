# Ubuntu/Mac 下载Webrtc，Chromium 源码

# 安装depot_tools

  1. Install the Chromium depot tools.
    1. 克隆depot_tools存储库：
git clone https://chromium.googlesource.com/chromium/tools/depot_tools.git
  1. 将depot_tools路径添加到系统环境变量中
   vi ~/.zshrc
   export PATH=$PATH:"/path/to/depot_tools"
   source ~/.zshrc
  1. Getting the Code
mkdir webrtc-checkout
cd webrtc-checkout
fetch --nohooks webrtc
gclient sync

# 自备梯子，例如shadowsocks

  1. git代理
git config --global http.proxy 'socks5://127.0.0.1:1080' 
git config --global https.proxy 'socks5://127.0.0.1:1080'
// 关闭代理
git config --global --unset http.proxy
git config --global --unset https.proxy
  1. ubuntu 用polipo代理http/https
    1. 安装polipo
sudo apt-get install polipo
  1. 配置polipo
vi /etc/polipo/config：

logSyslog = true
logFile = "/var/log/polipo/polipo.log"

socksParentProxy = "127.0.0.1:1080"
socksProxyType = socks5

chunkHighMark = 50331648
objectHighMark = 16384

serverMaxSlots = 64
serverSlots = 16
serverSlots1 = 32

proxyAddress = "0.0.0.0"
proxyPort = 8123
  1. 重启polipo服务：
sudo /etc/init.d/polipo restart
  1. 为终端配置http代理：
export http_proxy="http://127.0.0.1:8123/"
  1. 接着测试下能否科学上网：
curl www.google.com
  1. Mac 下不用polipo
export http_proxy=http://127.0.0.1:1080;
export https_proxy=http://127.0.0.1:1080;

# 出现的问题

  1. depot_tools 可能没有走这些代理（只是个提示完全可以不用管）
NOTICE: You have PROXY values set in your environment, but gsutilin depot_tools does not (yet) obey them.
Also, --no_auth prevents the normal BOTO_CONFIG environmentvariable from being used.
To use a proxy in this situation, please supply those settingsin a .boto file pointed to by the NO_AUTH_BOTO_CONFIG environmentvariable.

看上面的内容就知道大概该如何避免这些提示了。 在某个地方生成一个 .boto 文件内容如下

[Boto]
proxy = 127.0.0.1
proxy_port = 8123

export NO_AUTH_BOTO_CONFIG=/path/to/.boto

再去gclient sync 应该就没了
