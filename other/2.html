# git 多账户管理

# 生成多个ssh-key

  1. cd 到 根目录下的 .ssh 文件夹中，若没有则直接执行下面的代码会自动创建 .ssh文件夹
cd ~/.ssh
ssh-keygen -t rsa -C 'example.com'
  1. 填写生成文件的名字，默认是id_rsa,可以改成id_rsa_github等加以区分,然后全部默认enter
Enter file in which to save the key (/home/xxx/.ssh/id_rsa):id_rsa_github
  1. 在生成其他的ssh-key
ssh-keygen -t rsa -C 'example.com' (公司的gitlab，gitee等)
Enter file in which to save the key (/home/xxx/.ssh/id_rsa):id_rsa_gitlab(或着id_rsa_gitee)
  1. 修改config(没有就新建) vi config
#gitlab
Host gitlab.com
  HostName gitlab.com
  User exampleCompany.com
  PreferredAuthentications publickey
  IdentityFile ~/.ssh/id_rsa
#github
Host github.com
  HostName github.com
  User example.com
  PreferredAuthentications publickey
  IdentityFile ~/.ssh/id_rsa_github
#gitee
Host gitee.com
  HostName gitee.com
  User example.com
  PreferredAuthentications publickey
  IdentityFile ~/.ssh/id_rsa_gitee
  1. 去相对应的网站添加ssh-key
  2. 测试，出现Hi balabala 的就表示成功了
ssh -T git@github.com
ssh -T git@gitee.com
ssh -T git@gitlab.com
  1. repo提交账户不是想要的git的账户,git clone 下来的项目重新设置本地仓库的git配置
git config --local user.name xxx
git config --local user.email example@xx.com
