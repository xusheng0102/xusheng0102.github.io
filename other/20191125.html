# Ubuntu/Mac 终端使用代理

# Ubuntu

  1. 安装shadowsocks客户端
  sudo pip install shadowsocks
  1. 新建配置文件shadowsocks_config.json
{
    "server":"your server ip",
    "server_port":10000(your server port),
    "local_port":1080,
    "password":"your password",
    "timeout":300,
    "method":"aes-256-cfb"
}
  1. 启动
sslocal -c shadowsocks_config.json
  1. 使用
    1. chrome 下使用使用SwitchyOmega设置Chrome代理
    2. 终端 在.zshrc/.bash_profile 末尾添加
  alias proxyon="export all_proxy=socks5://127.0.0.1:1080"
  alias proxyoff="unset all_proxy"
  1. 终端使用代理(in terminal)
  source .zshrc(only once)
  proxyon

# Mac

  1. 安装
Mac 使用shadowsocks客户端即可
  1. 使用（同 Ubuntu 4）

# 错误处理

  1. sslocal -c shadowsocks_config.json
  AttributeError: /usr/lib/x86_64-linux-gnu/libcrypto.so.1.1: undefined symbol: EVP_CIPHER_CTX_cleanup

这个问题是由于在openssl1.1.0版本中，废弃了EVP_CIPHER_CTX_cleanup函数，如官网中所说：

EVP_CIPHER_CTX was made opaque in OpenSSL 1.1.0. As a result, EVP_CIPHER_CTX_reset() appeared and EVP_CIPHER_CTX_cleanup() disappeared.
EVP_CIPHER_CTX_init() remains as an alias for EVP_CIPHER_CTX_reset().



用vim打开文件：vim /usr/local/lib/python3.6/dist-packages/shadowsocks/crypto/openssl.py (多个python 目录的话，在pip对应的python目录下查找)
跳转到52行（shadowsocks2.8.2版本，其他版本搜索一下cleanup 全替换成 reset）
进入编辑模式
将第52行libcrypto.EVP_CIPHER_CTX_cleanup.argtypes = (c_void_p,)
改为libcrypto.EVP_CIPHER_CTX_reset.argtypes = (c_void_p,)
再次搜索cleanup（全文件共2处，此处位于111行），将libcrypto.EVP_CIPHER_CTX_cleanup(self._ctx)
改为libcrypto.EVP_CIPHER_CTX_reset(self._ctx)
保存并退出
启动shadowsocks服务：sslocal -c shadowsocks_config.json配置文件目录
sudo sslocal -c shadowlocal.json -d start
  1. 查看终端是否成功
curl ip.cn

或者

curl cip.cc
